Flink 流数据处理

Flink中的DataStream程序是定期在数据流上实现算子操作(eg: filtering, updating state, defining windows, aggregating). 数据源支持各种来源(eg: message queues, socket streams, files). 处理结果通过sinks将数据写入文件或标准输出.

WordCount程序示例:

import org.apache.flink.api.common.functions.FlatMapFunction;
import org.apache.flink.api.java.tuple.Tuple2;
import org.apache.flink.streaming.api.datastream.DataStream;
import org.apache.flink.streaming.api.environment.StreamExecutionEnvironment;
import org.apache.flink.streaming.api.windowing.time.Time;
import org.apache.flink.util.Collector;

public class WindowWordCount {

    public static void main(String[] args) throws Exception {

        StreamExecutionEnvironment env = StreamExecutionEnvironment.getExecutionEnvironment();

        DataStream<Tuple2<String, Integer>> dataStream = env
                //nc -lk 9999
                .socketTextStream("localhost", 9999)
                //split up the lines in pairs (2-tuples) containing: (word,1)
                .flatMap(new Splitter())
                .keyBy(0)//按第几个字段分组.
                .timeWindow(Time.seconds(5))//时间窗口5秒.
                .sum(1);//对第几个字段求和

        dataStream.print();

        env.execute("Window WordCount");
    }

    public static class Splitter implements FlatMapFunction<String, Tuple2<String, Integer>> {
        @Override
        public void flatMap(String sentence, Collector<Tuple2<String, Integer>> out) throws Exception {
            for (String word: sentence.split(" ")) {
                out.collect(new Tuple2<String, Integer>(word, 1));
            }
        }
    }

}

Data Sources

流从哪来.首先Flink提供大部分可用的数据源接口.当然你可以通过StreamExecutionEnvironment.addSource(sourceFunction)自定义数据源. 有2中方法实现自己的数据源:

  1. 通过实现ParallelSourceFunction接口.

  2. 通过继承RichParallelSourceFunction抽象类.

通过 StreamExecutionEnvironment 可以获取Flink内置的一些数据源:

  1. 文件相关:

    • readTextFile(path - 读取文本文件,该方法模式是使用TextInputFormat来一行一行的读取,并返回.

    • readFile(fileInputFormat, path 通过给定的InputFormat来实现读取文件.

    • readFile(fileInputFormat, path, watchType, interval, pathFilter, typeInfo 通过给定的InputFormat来读取文件

      • watchType 支持2种模式:1) 定期读取文件获取新数据. 2) 只读取一次就退出.

      • interval 时间间隔(ms) **pathFilter 文件过滤器.

      • typeInfo 关于输出流中元素类型的信息.

  2. Socket

    • socketTextStream - 通过socket获取流.

  3. Collection

    • fromCollection(Collection 通过java集合创建一个数据流.

    • fromCollection(Iterator, Class 通过iterator创建数据流.

    • `fromElements(T ` 通过一个序列对象创建数据流.

    • etc

  4. Custom

    • addSource 自定义数据源.

Data Sinks

sink指向数据流的流向. EG: files, sockets, external systems, or print.

  • writeAsText() TextOutputFormat,把数据以字符串的形式输出到文件.

  • writeAsCsv() CsvOutputFormat, 输出到CSV文件.

  • print()

  • writeUsingOutputFormat() FileOutputFormat,自定义output format.

  • writeToSocket() 把数据序列化写入socket

  • addSink 自定义sink

Iterations

Flink在DataStream中也是通过一个特定的可迭代的流（IterativeStream）来构建相关的迭代处理逻辑, 这一点跟DataSet提供的可迭代的数据集（IterativeDataSet）的是类似的.

流处理之迭代API

DataStream<Long> someIntegers = env.generateSequence(0, 1000);
// 基于输入流构建IterativeStream,这是一个迭代的起始,通常称之为迭代头
IterativeStream<Long> iteration = someIntegers.iterate();

// 我们指定一系列的转换操作用于表述在迭代过程中执行的逻辑(这里简单以map转换作为示例)
DataStream<Long> minusOne = iteration.map(new MapFunction<Long, Long>() {
  @Override
  public Long map(Long value) throws Exception {
    return value - 1 ;
  }
});

//作为迭代我们肯定需要有数据反馈给迭代头进行重复计算,
//所以我们从迭代过的流中过滤出符合条件的元素组成的部分流,我们称之为反馈流.
// 处理符合条件的数据流. 大于0的迭代过滤器
DataStream<Long> stillGreaterThanZero = minusOne.filter(new FilterFunction<Long>() {
  @Override
  public boolean filter(Long value) throws Exception {
    return (value > 0);
  }
});

//将反馈流反馈给迭代头就意味着一个迭代的完整逻辑的完成,那么它就可以"关闭"这个闭合的"环"了.
//通过调用IterativeStream的closeWith这一实例方法可以关闭一个迭代.
//传递给closeWith的数据流将会反馈给迭代头
iteration.closeWith(stillGreaterThanZero);

//我们需要过滤出继续向前分发的部分流,这个过滤转换其实定义的是"终止迭代"的逻辑条件,
//符合条件的元素将被分发给下游而不用于进行下一次迭代.
// 小于0的迭代过滤器.
DataStream<Long> lessThanZero = minusOne.filter(new FilterFunction<Long>() {
  @Override
  public boolean filter(Long value) throws Exception {
    return (value <= 0);
  }
});
