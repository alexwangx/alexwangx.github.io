Keyed 流

keyBy()

能够围绕流的属性之一对流进行分区通常非常有用, 以便将具有该属性的相同值的所有事件组合在一起. 例如, 假设我们想找到从每个网格单元中运行时间最长的出租车. 如果我们考虑的是SQL查询, 则意味着要使用 startCell 进行某种 GROUP BY , 而在Flink中, 这需要使用 keyBy(KeySelector) 完成

rides
  .flatMap(new NYCEnrichment())
  .keyBy("startCell")

每个keyBy都会进行 shuffle 操作, 从而对流进行重新分区. 通常代价非常高, 因为它涉及网络通信以及序列化和反序列化.

在上面的示例中, key 已由字段名称 "startCell" 指定. 这种类型的键选择具有一个缺点, 即编译器无法推断用于 key 的字段的类型, 因此Flink会将键值作为元组传递, 这可能很尴尬. 通常最好使用类型正确的KeySelector, eg:

rides
  .flatMap(new NYCEnrichment())
  .keyBy(
    new KeySelector<EnrichedRide, int>() {
      @Override
      public int getKey(EnrichedRide ride) throws Exception {
        return ride.startCell;
      }
    })

用lambda可以更简洁地表达:

rides
  .flatMap(new NYCEnrichment())
  .keyBy(ride -> ride.startCell)

Keyed 流上的聚合

此代码段创建一个新的元组流, 其中包含每个骑乘结束事件的 startCell 和持续时间（以分钟为单位）:

DataStream<Tuple2<Integer, Minutes>> minutesByStartCell = enrichedNYCRides
  .flatMap(new FlatMapFunction<EnrichedRide, Tuple2<Integer, Minutes>>() {
    @Override
    public void flatMap(EnrichedRide ride,
              Collector<Tuple2<Integer, Minutes>> out) throws Exception {
      if (!ride.isStart) {
        Interval rideInterval = new Interval(ride.startTime, ride.endTime);
        Minutes duration = rideInterval.toDuration().toStandardMinutes();
        out.collect(new Tuple2<>(ride.startCell, duration));
      }
    }
  });

现在, 我们可以生成一个流, 其中仅包含每个 startCell 迄今为止最长的ride. 可以使用多种方式表示要用作关键字的字段. 前面我们看到了一个 EnrichedRide POJO的示例, 在其中我们通过使用其名称指定了要使用的字段. 在这个例子下, 我们将使用Tuple2对象, 并在元组中提供从0开始的索引.

minutesByStartCell
  .keyBy(0) // startCell
  .maxBy(1) // duration
  .print();

在输出流中, 每次持续时间达到新的最大值时, 我们都会看到每个键的记录. 比如 50797,8M:

...
4> (64549,5M)
4> (46298,18M)
1> (51549,14M)
1> (53043,13M)
1> (56031,22M)
1> (50797,6M)
...
1> (50797,8M)
...
1> (50797,11M)
...
1> (50797,12M)

(隐含)状态

这是我们看到的第一个有状态流的示例. 尽管状态是透明处理的, 但是Flink必须跟踪每个不同 key 的最大持续时间. 每当状态涉及到您的应用程序时, 您都应该考虑状态可能会变大. 只要 Key 空间是无界的, 那么 Flink 将需要的状态量也是那么多. 在使用流时, 通常考虑在有限窗口而不是整个流上进行聚合来思考.

reduce() 和其他聚合器

上面使用的 maxBy() 只是Flink的 KeyedStream 上可用的许多聚合函数的一个示例. 还有一个更通用的 reduce() 函数, 可用于实现自己的自定义聚合.
