Multiple regions have the same startkey

  1. HRegionInfo was null in tablename.

    • 这个问题的根本是有两个起始 key 相同的 region导致.

    • 解决办法是找到这两个 region, 然后在 meta 表里面把这条数据删除掉. 重新 repair 修复一下.

      2019-02-27 19:18:30,458 WARN  [main] util.HBaseFsck: Naming new problem group: fcd4e
ERROR: (region issues_table,fcd4e,1551258254806.e4f2ba40903670cd787e0bd1397796a5.) Multiple regions have the same startkey: fcd4e
ERROR: (region issues_table,fcd4e,1548789303954.1851d00bf771171a2e63caf29a55af3f.) Multiple regions have the same startkey: fcd4e
ERROR: (regions issues_table,fcd4e,1548789303954.1851d00bf771171a2e63caf29a55af3f. and issues_table,fe6c3,1551258254806.5141a789e41912c0f56e366de240bd60.) There is an overlap in the region chain.

  2. 解决办法:

    # 删除没上线的region meta信息
delete 'hbase:meta','issues_table,fcd4e,1548789303954.1851d00bf771171a2e63caf29a55af3f.','info:regioninfo'

  3. 删除hdfs中region数据:

    hdfs dfs -rmr /hbase/data/default/issues_table/1851d00bf771171a2e63caf29a55af3f

  4. 执行 hbase hbck issues_table -repair 解决问题
